## Users

ทีมการตลาดภายในของ nosetea (เช่นเจ้าของอีเมล marketingai03@nosetea.com) ที่ดูแลหลายแบรนด์/หลายแคมเปญ ผู้ใช้ไม่จำเป็นต้องเป็นสายเทคนิค — เข้ามาทำงานเป็นช่วงๆ เพื่อสร้าง short link/QR, เปลี่ยนปลายทาง, ดูผลคลิก/สแกน, จัดหน้า bio ของแบรนด์ และจัดระเบียบตามแบรนด์ ฝั่งจัดการใช้เดสก์ท็อปเป็นหลัก (ลากจัดกลุ่ม) ส่วนหน้า bio public คนปลายทางเปิดจากมือถือ (IG/TikTok) UI ภาษาไทย

## Product Purpose

แพลตฟอร์มจัดการ **Dynamic Links** ภายในทีม (แนว Bitly แต่ใช้เอง) — **short link เป็นแกนกลาง** (`/r/{code}` เปลี่ยนปลายทางได้ตลอด) โดยมี feature ต่อยอดจากลิงก์:

1. **QR code** — gen จาก short link พร้อมปรับแต่ง (สี/dot/โลโก้) พิมพ์ครั้งเดียวใช้ตลอด
2. **Analytics** — คลิก/สแกนต่อลิงก์: อุปกรณ์, referrer, ประเทศ, กราฟเวลา, export CSV
3. **Bio pages (แบบ Linktree)** — หน้า `/p/{handle}` รวมหลายลิงก์ต่อแบรนด์ ทุกปุ่มชี้ผ่าน short link จึงได้ analytics อัตโนมัติ
4. **จัดระเบียบหลายแบรนด์** — โฟลเดอร์/กลุ่ม (drag & drop), ค้นหา, แท็ก
5. **Custom domain ต่อแบรนด์** (แผนถัดไป) — เช่น `go.tanicha.co/summer`

ความสำเร็จ = ทีมจัดการลิงก์+QR+หน้า bio ของหลายแบรนด์จากที่เดียว, แก้ปลายทางแคมเปญได้ทันทีโดยของที่แชร์/พิมพ์ไปแล้วยังใช้ได้, และเห็นผลคลิกที่นำไปตัดสินใจต่อได้

**ขอบเขตที่ตัดโดยตั้งใจ** (ใช้ภายในทีมเดียว): ไม่มีระบบ login/users/workspaces และไม่มี public API/API keys — ดูเหตุผลและเงื่อนไขใน `ROADMAP.md`

Roadmap ฉบับเต็ม: `ROADMAP.md` (Phase A Link-first → B Management → C Analytics v2 → E Bio Pages → D Custom Domains → G Performance)

## Brand Personality

เครื่องมือที่ "หลบทาง" ให้งาน — ความชัดและใช้ง่ายมาก่อนการแสดงออกทางดีไซน์ เรียบ นิ่ง เชื่อถือได้ ไม่ต้องมีบุคลิกจัดจ้าน เป้าหมายเดียวคือทำให้ UX ดีขึ้นและงานเสร็จเร็วขึ้น น้ำเสียง UX copy: ตรงไปตรงมา กระชับ เป็นภาษาไทยที่เข้าใจทันที

## Anti-references

- **SaaS/AI-generic โหลๆ** — การ์ดเหมือนกันเรียงกันทั้งหน้า, gradient ฟุ้ง, gradient text, hero-metric template, eyebrow ตัวเล็ก uppercase เหนือทุกหัวข้อ, glassmorphism ประดับ ทุกอย่างที่ดู "AI สร้าง"
- **รก / แน่นเกินไป** — ปุ่มเยอะ, ข้อมูลอัดแน่น, ลำดับความสำคัญไม่ชัด, ผู้ใช้ต้องเพ่งหาว่าจะทำอะไรต่อ

## Design Principles

1. **Usability over expression** — ถ้าต้องเลือกระหว่างสวยกับใช้ง่าย เลือกใช้ง่าย ดีไซน์รับใช้งาน ไม่ใช่โชว์ตัวเอง
2. **Link-first, dynamic-first** — short link คือแกน ทุก feature (QR, bio page, analytics) ต่อจากลิงก์ คุณค่าหลัก (แก้ปลายทางได้โดยลิงก์/QR ที่แชร์ไปแล้วไม่เปลี่ยน) ต้องสื่อและเข้าถึงง่ายในทุกจุด
3. **หลายแบรนด์ในที่เดียว** — การจัดระเบียบ (โฟลเดอร์/ฟิลเตอร์/ค้นหา) เป็นพลเมืองชั้นหนึ่ง ไม่ใช่ของแถม
4. **Analytics ที่ actionable** — ตัวเลขต้องอ่านง่ายและนำไปตัดสินใจได้ ไม่ใช่ dashboard ตัวเลขลอยๆ
5. **ลด cognitive load** — แต่ละหน้าเห็นชัดว่า "ทำอะไรต่อ" ลดของรก ต้านตรงกับ anti-reference เรื่องความแน่น

## Accessibility & Inclusion

เป้าหมาย WCAG 2.1 AA: body text contrast ≥ 4.5:1 (ระวัง neutral-400/500 บนพื้นเข้มให้ผ่านจริง), large text ≥ 3:1, focus state มองเห็นได้, ใช้งานคีย์บอร์ดได้, ไม่พึ่งสีอย่างเดียวในการสื่อสถานะ (เปิด/ปิด, เป้า drop), รองรับ `prefers-reduced-motion` เมื่อเพิ่ม motion
